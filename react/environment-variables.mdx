# Managing Environment Variables

Modern React applications rely on API endpoints, feature flags, and third-party integrations.  
VectorIHub allows you to securely manage these using **environment variables** â€” without exposing sensitive data in your repository.

<Note>
Environment variables are securely injected during the build and deployment process and are not stored in your source code.
</Note>

## Adding Environment Variables

Follow these steps to add environment variables to your React project:

<Steps>
  <Step title="Open Project Dashboard">
    Go to your project inside the **VectorIHub Dashboard**.
  </Step>

  <Step title="Navigate to Environment Settings">
    Open:
    <Info>Settings â†’ Environment Variables</Info>
  </Step>

  <Step title="Add Key-Value Pairs">
    Add your environment variables such as API URLs, public keys, or feature flags.

    ```bash
    Key: VITE_API_URL
    Value: https://api.vectorihub.com
    ```

    <Check>
    Your variables are now securely stored and will be injected during the build.
    </Check>
  </Step>
</Steps>

## Accessing Variables in React (Vite)

For Vite-based React apps, environment variables are accessed using `import.meta.env`.

<CodeGroup>
```jsx App.jsx
const apiUrl = import.meta.env.VITE_API_URL;

export default function App() {
  return (
    <div>
      <h1>VectorIHub React App ðŸš€</h1>
      <p>API Endpoint: {apiUrl}</p>
    </div>
  );
}
```
</CodeGroup>

<Tip> Vite requires environment variables to be prefixed with `VITE_` to be exposed to the browser. </Tip>

## Accessing Variables in Create React App (CRA)

If you are using Create React App, use the REACT_APP_ prefix.

<CodeGroup> 
  ```jsx App.js 
  const apiUrl = process.env.REACT_APP_API_URL;

  function App() {
    return (
    <div>
    <h1>Connected to: {apiUrl}</h1>
    </div>
    );
  }

  export default App;
  ```
</CodeGroup>

## Updating Environment Variables

Environment variable updates are applied during the next deployment cycle.

<AccordionGroup>
  <Accordion title="How updates are applied">
    - Modify or add variables in the dashboard  
    - Trigger a redeploy or push new code  
    - VectorIHub rebuilds the React app with updated variables  
  </Accordion>

  <Accordion title="When should you redeploy?">
    Redeploy whenever you:
    - Change API endpoints  
    - Update public configuration values  
    - Add new third-party integrations  
  </Accordion>
</AccordionGroup>

<Badge color="amber">Requires Redeploy</Badge>
